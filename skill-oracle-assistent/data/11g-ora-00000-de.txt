ORA-00000: normaler, erfolgreicher Abschluss
Ursache: Normaler Ausgang.
Aktion: Keine
ORA-00001: eindeutige Einschränkung ( Zeichenfolge . Zeichenfolge ) verletzt
Ursache: Eine UPDATE- oder INSERT-Anweisung hat versucht, einen doppelten Schlüssel einzufügen. Wenn Trusted Oracle im DBMS-MAC-Modus konfiguriert ist, wird diese Meldung möglicherweise angezeigt, wenn auf einer anderen Ebene ein doppelter Eintrag vorhanden ist.
Aktion: Entfernen Sie entweder die einmalige Einschränkung oder fügen Sie den Schlüssel nicht ein.
ORA-00017: Sitzung angefordert, um Trace-Ereignis zu setzen
Ursache: Die aktuelle Sitzung wurde aufgefordert, ein Trace-Ereignis durch eine andere Sitzung festzulegen.
Aktion: Dies wird intern verwendet; keine Aktion ist erforderlich.
ORA-00018: Maximale Anzahl der Sitzungen überschritten
Ursache: Alle Sitzungsstatusobjekte werden verwendet.
Aktion: Erhöhen Sie den Wert des Initialisierungsparameters SESSIONS.
ORA-00019: Maximale Anzahl der Sitzungslizenzen überschritten
Ursache: Alle Lizenzen werden verwendet.
Aktion: Erhöhen Sie den Wert des Initialisierungsparameters LICENSE MAX SESSIONS.
ORA-00020: Maximale Anzahl der Prozesse ( String ) überschritten
Ursache: Alle Prozessstatusobjekte werden verwendet.
Aktion: Erhöhen Sie den Wert des Initialisierungsparameters PROCESSES.
ORA-00021: Sitzung an einen anderen Prozess angehängt; Sitzung kann nicht gewechselt werden
Ursache: Die Benutzersitzung wird derzeit von anderen Benutzern verwendet.
Aktion: Wechseln Sie nicht zu einer Sitzung, die an einen anderen Prozess angehängt ist.
ORA-00022: ungültige Sitzungs-ID; Zugriff abgelehnt
Ursache: Entweder ist die angegebene Sitzung nicht vorhanden oder der Aufrufer hat nicht die Berechtigung, darauf zuzugreifen.
Aktion: Geben Sie eine gültige Sitzungs-ID an, auf die Sie Zugriff haben, entweder Sie besitzen sie oder Sie haben die Berechtigung CHANGE_USER.
ORA-00023: Sitzungsreferenzen verarbeiten privaten Speicher; Sitzung kann nicht getrennt werden
Ursache: Es wurde versucht, die aktuelle Sitzung zu trennen, wenn sie Verweise zur Verarbeitung von privatem Speicher enthält.
Aktion: Eine Sitzung kann Verweise auf Prozessspeicher (PGA) enthalten, wenn sie eine offene Netzwerkverbindung, einen sehr großen Kontextbereich oder Betriebssystemprivilegien aufweist. Um das Trennen zu ermöglichen, müssen Sie möglicherweise die Datenbankverbindungen und / oder Cursor der Sitzung schließen. Das Trennen einer Sitzung mit Betriebssystemprivilegien ist immer nicht zulässig.
ORA-00024: Logins von mehr als einem Prozess sind im Single-Process-Modus nicht erlaubt
Ursache: Der Versuch, sich mehrmals aus verschiedenen Prozessen für ORACLE anzumelden, wurde im Einzelprozessmodus gestartet.
Aktion: Abmelden von dem anderen Prozess.
ORA-00025: Zuweisung der Zeichenfolge fehlgeschlagen
Ursache: Nicht genügend Arbeitsspeicher.
Aktion: Neustart mit größerem SGA-Heap.
ORA-00026: fehlende oder ungültige Sitzungs-ID

Aktion: Erneut mit einer gültigen Sitzungs-ID versuchen.
ORA-00027: Aktuelle Sitzung kann nicht beendet werden

Aktion: Keine
ORA-00028: Deine Sitzung wurde getötet
Ursache: Ein privilegierter Benutzer hat Ihre Sitzung beendet und Sie sind nicht mehr bei der Datenbank angemeldet.
Aktion: Melden Sie sich erneut an, wenn Sie weiterarbeiten möchten.
ORA-00029: Sitzung ist keine Benutzersitzung
Ursache: Die in einem Befehl ALTER SYSTEM KILL SESSION angegebene Sitzungs-ID war keine Benutzersitzung (z. B. rekursiv usw.).
Aktion: Erneut mit einer Benutzersitzungs-ID versuchen.
ORA-00030: Benutzersitzungs-ID existiert nicht.
Ursache: Die Benutzersitzungs-ID ist nicht mehr vorhanden, wahrscheinlich weil die Sitzung abgemeldet wurde.
Aktion: Verwenden Sie eine gültige Sitzungs-ID.
ORA-00031: Sitzung wurde zum Beenden markiert
Ursache: Die in einem ALTER SYSTEM KILL SESSION-Befehl angegebene Sitzung kann nicht sofort beendet werden (weil sie bei einem Netzwerkvorgang zurückgesetzt oder blockiert wird), wurde jedoch zum Beenden markiert. Dies bedeutet, dass es so bald wie möglich getötet wird, nachdem seine aktuelle unterbrechungsfreie Operation durchgeführt wurde.
Aktion: Es ist keine Aktion erforderlich, damit die Sitzung beendet wird. Weitere Ausführungen des Befehls ALTER SYSTEM KILL SESSION in dieser Sitzung können jedoch dazu führen, dass die Sitzung früher beendet wird.
ORA-00032: ungültiges Sitzungsmigrationskennwort
Ursache: Das Sitzungsmigrationskennwort, das bei einem Sitzungserstellungsaufruf angegeben wurde, war ungültig (wahrscheinlich zu lang).
Aktion: Erneut mit einem gültigen Passwort (weniger als 30 Zeichen) versuchen.
ORA-00033: Die aktuelle Sitzung hat ein leeres Migrationskennwort
Ursache: Es wurde versucht, die aktuelle Sitzung zu trennen oder zu klonen, und es ist ein leeres Migrationskennwort vorhanden. Das ist nicht erlaubt.
Aktion: Erstellen Sie die Sitzung mit einem nicht leeren Migrationskennwort.
ORA-00034: Die aktuelle PL / SQL-Sitzung kann nicht mit einem String versehen werden
Ursache: Es wurde versucht, eine COMMIT- oder Rollback-Operation von einem PL / SQL-Objekt (Prozedur, Funktion, Paket) in einer Sitzung auszuführen, für die dies deaktiviert wurde (durch 'Sitzung in COMBINATION' deaktivieren).
Aktion: Aktivieren Sie in dieser Sitzung Commits von PL / SQL oder versuchen Sie nicht, Commit oder Rollback in PL / SQL zu verwenden, wenn sie in der aktuellen Sitzung deaktiviert sind.
ORA-00036: Maximale Anzahl rekursiver SQL-Level ( String ) überschritten
Ursache: Es wurde versucht, mehr als die angegebene Anzahl rekursiver SQL-Ebenen zu durchlaufen.
Aktion: Entfernen Sie das rekursive SQL, möglicherweise einen rekursiven Trigger.
ORA-00037: kann nicht zu einer Sitzung wechseln, die zu einer anderen Servergruppe gehört
Ursache: Es wurde versucht, zu einer Sitzung in einer anderen Servergruppe zu wechseln. Das ist nicht erlaubt.
Aktion: Stellen Sie sicher, dass der Server zu einer Sitzung wechselt, die zu seiner Servergruppe gehört.
ORA-00038: Sitzung kann nicht erstellt werden: Servergruppe gehört einem anderen Benutzer
Ursache: Es wurde versucht, eine nicht migrierbare Sitzung in einer Servergruppe zu erstellen, die einem anderen Benutzer gehört.
Aktion: Eine Servergruppe gehört dem ersten Benutzer, der sich bei einem Server in der Servergruppe im nicht migrierbaren Modus anmeldet. Alle nachfolgenden nicht migrierbaren Modus-Logins müssen von dem Benutzer vorgenommen werden, der die Servergruppe besitzt. Um eine andere Benutzeranmeldung im nicht migrierbaren Modus zu erhalten, muss die Eigentümerschaft der Servergruppe geändert werden. Dies kann erreicht werden, indem alle aktuellen Sitzungen abgemeldet und von allen vorhandenen Servern in der Servergruppe getrennt werden und der neue Benutzer sich als neuer Besitzer anmeldet.
ORA-00039: Fehler während der periodischen Aktion

Aktion: Überprüfen Sie den Fehlerstapel auf detaillierte Fehlerinformationen.

ORA-00043: Remote-Vorgang ist fehlgeschlagen
Ursache: Die Ausführung der Interinstanzoperation ist fehlgeschlagen.
Aktion: Überprüfen Sie den Status der Zielinstanz. Die Operation wurde möglicherweise teilweise ausgeführt. Überprüfen Sie das Ergebnis des beabsichtigten Vorgangs.
ORA-00044: timed_statistics muss TRUE sein, wenn statistics_level nicht BASIC ist
Ursache: Der Benutzer hat versucht, timed_statistics auf FALSE zu setzen, wenn statistics_level nicht BASIC ist. timed_statistics muss TRUE sein, um aussagekräftige Statistiken zu sammeln, wenn statistics_level auf TYPICAL oder ALL gesetzt ist.
Aktion: Setzen Sie timed_statistics entweder auf TRUE oder setzen Sie statistics_level auf BASIC.
ORA-00050: Beim Ermitteln einer Warteschlange ist ein Betriebssystemfehler aufgetreten
Ursache: Die für eine Oracle-Enqueue erforderlichen Betriebssystemressourcen konnten nicht abgerufen werden. Dies ist normalerweise das Ergebnis eines zu niedrigen Benutzerkontingents des Betriebssystems.
Aktion: Suchen Sie in Ihrer Systemdokumentation nach dem Betriebssystemfehler und führen Sie die erforderliche Aktion aus.

Aktion: Siehe weitere Fehlermeldungen für weitere Details des Problems gezeigt.
ORA-00752: Wiederherstellung entdeckte ein verlorenes Schreiben eines Datenblockes
Ursache: Ein Schreiben eines Datenblocks auf der Speicher werden wurde während des normalen Datenbankbetrieb oder in der primären Datenbank verloren.
Aktion: Rückblick oder Wiederherstellung von Backup der Datenbank , wie in dem Backup- und Recovery - Handbuch beschrieben.
ORA-00820: Sollwert von SGA_MAX_SIZE zu klein ist , muss zumindest sein String M
Ursache: Der angegebene Wert von SGA_MAX_SIZE ist zu klein für die SGA all notwendigen SGA - Komponenten wie die Protokollpuffer, Pufferpools, gemeinsamen Pool gerecht zu werden , usw.
Aktion: SGA_MAX_SIZE auf den empfohlenen Wert einstellen oder die Werte aller SGA Komponente Größenparameter reduzieren Sie angegeben haben.
ORA-00821: Sollwert von SGA_TARGET String M zu klein ist , muss zumindest sein String M
Ursache: Der angegebene Wert von SGA_TARGET ist zu klein für die SGA all notwendigen SGA - Komponenten wie die Protokollpuffer, Pufferpools, gemeinsamen Pool gerecht zu werden , usw.
Aktion: Stellen Sie SGA_TARGET auf den empfohlenen Wert oder die Werte aller Größenparameter SGA Komponente reduzieren Sie angegeben haben.
ORA-00822: MMAN Prozess mit Fehlern beendet
Ursache: Der Memory - Management - Prozess gestorben.
Aktion: Warmstart Instanz
ORA-00823: Sollwert von SGA_TARGET größer als SGA_MAX_SIZE
Ursache: Der angegebene Wert von SGA_TARGET größer als SGA_MAX_SIZE.
Aktion: Erhöhen Sie SGA_MAX_SIZE mit SGA_TARGET oder Abnahme SGA_TARGET passen mit sga_maxsize passen.
ORA-00824: kann nicht SGA_TARGET oder MEMORY_TARGET aufgrund bestehender interner Einstellungen
Ursache: Kann nicht gesetzt SGA_TARGET oder MEMORY_TARGET aufgrund der aktuellen Parametereinstellungen.
Aktion: Überprüfen Sie die anderen Fehlermeldungen für weitere Informationen.
ORA-00825: kann nicht eingestellt DB_BLOCK_BUFFERS wenn SGA_TARGET oder MEMORY_TARGET gesetzt
Ursache: SGA_TARGET oder MEMORY_TARGET Set mit DB_BLOCK_BUFFERS gesetzt.
Aktion: Stellen Sie nicht SGA_TARGET, MEMORY_TARGET oder eine neue Cache - Parameter verwenden, und verwenden Sie DB_BLOCK_BUFFERS nicht , die ein alter Cache - Parameter ist.
ORA-00826: kann nicht eingestellt SGA_TARGET oder MEMORY_TARGET für eine ASM-Instanz
Ursache: SGA_TARGET oder MEMORY_TARGET für eine ASM - Instanz festgelegt.
Aktion: Sie setzen SGA_TARGET oder MEMORY_TARGET nicht.
ORA-00827: konnte nicht SGA_TARGET zu bestimmten Wert schrumpfen
Ursache: Versuchte die SGA auf den angegebenen Wert zu schrumpfen , aber nicht gelang , weil die SGA - Komponenten nicht , wie sie auf den kleinsten Größen bereits waren geschrumpft werden.
Aktion: Sie SGA_TARGET auf einen Wert unter dem aktuellen Wert nicht festgelegt , ohne zunächst die einzelnen Komponenten SGA schrumpft.
ORA-00828: angegebener Wert von shared_pool_reserved_size nicht mit internen Einstellungen
Ursache: Nicht imstande , shared_pool_reserved_size zu festgelegten Wert , wenn SGA_TARGET Satz zu setzen, sei es, weil der angegebene Wert zu klein ist , oder weil es zu groß für die aktuelle interne Größe des gemeinsamen Pool. Weitere Details finden Sie in dem Alert - Protokoll gefunden werden.
Aktion: Wenn möglich, nicht gesetzt shared_pool_reserved_size ohne shared_pool_size Einstellung , wenn SGA_TARGET gesetzt. Untersuchen Sie das Alarmprotokoll für Informationen über aktuelle interne Größe des gemeinsam genutzten Pools und gültigen Wertebereich für shared_pool_reserved_size.
ORA-00830: nicht STATISTICS_LEVEL zur Grundausstattung mit SGA auto-tune aktiviert
Ursache: Der Benutzer hat versucht STATISTICS_LEVEL zu BASIC - Set mit SGA auto-tune aktiviert , die getan werden kann , nicht weil Auto-Tune SGA kann nicht mit STATISTICS_LEVEL zur Grundausstattung arbeiten.
Aktion: Deaktivieren Sie Auto-Tune - SGA und versuchen Sie es erneut die STATISTICS_LEVEL in Grundeinstellung.
ORA-00832: kein Streams Pool erstellt und kann nicht automatisch erstellen
Ursache: Eine Datenbank - Funktion , die STREAMS muss SGA verwendet wurde, jedoch wurde der STREAMS_POOL_SIZE Parameter nicht definiert und der Wert von db_cache_size war zu klein , um eine automatische Übertragung von SGA zu dem Streams Pool aus dem Puffer - Cache zu ermöglichen.
Aktion: Bitte stellen Sie die Parameter STREAMS_POOL_SIZE oder SGA_TARGET eingestellt.
ORA-00837: Sollwert von MEMORY_TARGET größer als MEMORY_MAX_TARGET
Ursache: Der angegebene Wert von MEMORY_TARGET war größer als MEMORY_MAX_TARGET.
Aktion: Stellen Sie MEMORY_TARGET weniger als oder gleich MEMORY_MAX_TARGET.
ORA-00838: Sollwert von MEMORY_TARGET zu klein ist , muss zumindest sein String M
Ursache: Der angegebene Wert von MEMORY_TARGET war kleiner als die Summe der angegebenen Werte für SGA_TARGET und PGA_AGGREGATE_TARGET.
Aktion: Stellen Sie MEMORY_TARGET mindestens den empfohlenen Wert.
ORA-00839: SGA_TARGET kann nicht auf den angegebenen Wert geändert werden
Ursache: Der angegebene Wert von SGA_TARGET war nicht kompatibel mit den aktuellen Einstellungen von PGA_AGGREGATE_TARGET und MEMORY_TARGET / MEMORY_MAX_TARGET.
Aktion: Stellen Sie SGA_TARGET nicht mehr als die Differenz von MEMORY_TARGET / MEMORY_MAX_TARGET und PGA_AGGREGATE_TARGET zu sein. Achten Sie auch darauf , dass der aktuelle PGA - Speicher für Nicht-Autoworkareas verwendet wird , ist gering.
ORA-00840: PGA_AGGREGATE_TARGET kann nicht auf den angegebenen Wert geändert werden
Ursache: Der angegebene Wert von PGA_AGGREGATE_TARGET war nicht kompatibel mit den aktuellen Einstellungen von SGA_TARGET und MEMORY_TARGET / MEMORY_MAX_TARGET.
Aktion: Set PGA_AGGREGATE_TARGET nicht mehr als die Differenz von MEMORY_TARGET / MEMORY_MAX_TARGET und SGA_TARGET zu sein.
ORA-00841: MEMORY_TARGET auf diesem System nicht unterstützt
Ursache: MEMORY_TARGET Parameter wurden auf diesem System nicht unterstützt , da shmfs deaktiviert wurde.
Aktion: Siehe für eine Liste der unterstützten Systeme zur Dokumentation.
ORA-00842: DB_BLOCK_BUFFERS kann nicht mit SGA_TARGET oder MEMORY_TARGET eingestellt werden
Ursache: DB_BLOCK_BUFFERS war unvereinbar mit SGA_TARGET und MEMORY_TARGET.
Aktion: Entfernen Sie die inkompatiblen Parametereinstellung.
ORA-00843: Parameter nicht MEMORY_MAX_TARGET berücksichtigt
Ursache: Der Parameter war größer als MEMORY_MAX_TARGET.
Aktion: Überprüfen Sie die anderen Fehlermeldungen für weitere Informationen über die Parameter. Stellen Sie den Parameter auf einen niedrigeren Wert als MEMORY_MAX_TARGET.
ORA-00844: Parameter nicht MEMORY_TARGET berücksichtigt
Ursache: Der Parameter war größer als MEMORY_TARGET.
Aktion: Überprüfen Sie die anderen Fehlermeldungen für weitere Informationen über die Parameter. Stellen Sie den Parameter auf einen niedrigeren Wert als MEMORY_TARGET.
ORA-00845: MEMORY_TARGET auf diesem System nicht unterstützt
Ursache: Der MEMORY_TARGET Parameter wurde nicht auf diesem Betriebssystem nicht unterstützt oder / dev / shm wurde nicht richtig auf Linux bemessen.
Aktion: Siehe für eine Liste der unterstützten Betriebssysteme bis hin zur Dokumentation. Oder Größe / dev / shm zumindest die SGA_MAX_SIZE auf jeder Oracle - Instanz auf dem System ausgeführt werden.
ORA-00846: konnte nicht MEMORY_TARGET zu bestimmten Wert schrumpfen
Ursache: Versuchte MEMORY_TARGET auf den angegebenen Wert zu schrumpfen , aber nicht erfolgreich , da die PGA und SGA - Komponenten nicht geschrumpft werden.
Aktion: Stellen Sie nicht MEMORY_TARGET auf einen Wert unter dem aktuellen Wert , ohne zunächst die einzelnen Speicherkomponenten zu schrumpfen.
ORA-00847: MEMORY_TARGET / MEMORY_MAX_TARGET und LOCK_SGA kann nicht zusammen gesetzt werden
Ursache: MEMORY_TARGET / MEMORY_MAX_TARGET wurde auf einen Wert ungleich Null gesetzt und LOCK_SGA wurde auch festgelegt
Aktion: Sie setzen MEMORY_TARGET oder MEMORY_MAX_TARGET nicht , wenn LOCK_SGA auf TRUE gesetzt ist.
ORA-00848: STATISTICS_LEVEL kann nicht in BASIC mit SGA_TARGET oder MEMORY_TARGET eingestellt werden
Ursache: STATISTICS_LEVEL von BASIC war unvereinbar mit SGA_TARGET und MEMORY_TARGET.
Aktion: Befestigen Sie die inkompatiblen Parametereinstellung. Stellen Sie STATISTICS_LEVEL auf TYPICAL oder ALL SGA_TARGET oder MEMORY_TARGET zu verwenden.
ORA-00849: SGA_TARGET String kann nicht auf mehr als MEMORY_MAX_TARGET eingestellt werden Zeichenfolge .
Ursache: SGA_TARGET Wert war mehr als MEMORY_MAX_TARGET Wert.
Aktion: Stellen Sie SGA_TARGET auf weniger als MEMORY_MAX_TARGET.
ORA-00850: PGA_AGGREGATE_TARGET String kann nicht auf mehr als MEMORY_MAX_TARGET eingestellt werden Zeichenfolge .
Ursache: PGA_AGGREGATE_TARGET Wert war mehr als MEMORY_MAX_TARGET Wert.
Aktion: Set PGA_AGGREGATE_TARGET auf weniger als MEMORY_MAX_TARGET.
ORA-00851: SGA_MAX_SIZE String kann nicht auf mehr als MEMORY_TARGET eingestellt werden Zeichenfolge .
Ursache: SGA_MAX_SIZE Wert war mehr als MEMORY_TARGET Wert.
Aktion: Stellen Sie SGA_MAX_SIZE auf weniger als MEMORY_TARGET.
